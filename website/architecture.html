Architecture

Synth explained using only the seven public concepts.

The public flow

Idea → Mission → Planning → Approval → Commit → Execution → Events → State → Replay

Step by step

  1. Idea — Someone describes what they want to achieve.
  2. Mission — The idea becomes a strategic goal.
  3. Planning — Evidence is gathered and expeditions are defined.
  4. Approval — The plan is reviewed and approved.
  5. Commit — The approved plan is committed.
  6. Execution — Actions are requested and, if allowed, recorded.
  7. Events — Every action is appended as an immutable record.
  8. State — The current picture is derived from events.
  9. Replay — The event log is replayed to prove correctness.

What is not shown

Internal components are intentionally omitted from public architecture explanations. Operators do not need to know them to use Synth correctly.
