# Bootstrap Manifest

The Bootstrap Manifest is an agent-readable description of a Synth project. It is generated by `synth init` and lives at `.synth/manifest.json`.

## Purpose

- Let an AI agent discover available commands without reading architecture documentation.
- Provide a stable contract between the project and the agent.
- Surface the public vocabulary, project layout, and capabilities.

## Schema

```json
{
  "schema": "synth-bootstrap-manifest-v1",
  "version": "2.0.0",
  "projectName": "My Project",
  "root": "/path/to/project",
  "generatedAt": "2026-07-12T00:00:00.000Z",
  "commands": [
    { "name": "init", "description": "Initialize the current directory as a Synth project" }
  ],
  "capabilities": ["repository", "github", "tdd", "bdd"],
  "layout": {
    "docs": "docs/",
    "generatedDocs": "docs/generated/",
    "examples": "examples/",
    "data": ".synth/data/",
    "proof": "proof/",
    "src": "src/",
    "tests": "tests/",
    "scripts": "scripts/",
    "website": "website/"
  },
  "publicVocabulary": ["Mission", "Expedition", "Evidence", "Plan", "Event", "State", "Replay"],
  "govern": "npm run govern",
  "quickStart": "synth init && synth docs generate && npm run govern"
}
```

## Fields

| Field | Type | Description |
|---|---|---|
| `schema` | string | Manifest schema version. |
| `version` | string | Installed Synth version. |
| `projectName` | string | Human-readable project name. |
| `root` | string | Absolute path to project root. |
| `generatedAt` | string | ISO-8601 timestamp. |
| `commands` | array | Available Synth CLI commands. |
| `capabilities` | array | Adapter names available in this project. |
| `layout` | object | Key directories relative to project root. |
| `publicVocabulary` | array | The seven public concepts. |
| `govern` | string | Command to run full governance. |
| `quickStart` | string | One-line quick start command. |

## Usage

An AI agent should read `.synth/manifest.json` before operating on the project. It can then:

1. Confirm the project is a Synth project.
2. Discover supported commands.
3. Understand the public vocabulary.
4. Locate generated documentation, examples, and proof artifacts.

## Generation

Run:

```bash
synth init
```

To specify a project name:

```bash
synth init --name "My Project"
```

## Stability

The manifest is generated. Do not edit it by hand. Re-run `synth init` to update it after upgrading Synth.
