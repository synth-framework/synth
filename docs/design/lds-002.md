# Mission Studio Design System (LDS-002 v3)

> **Mission Studio is the primitive of SYNTH's public interface.** The homepage is one projection of Mission Studio. Future surfaces — desktop, VS Code extension, mobile — will consume the same runtime contract and render through this design system.

**Version:** 3.0.0
**Theme:** Light only

---

## Thesis

Do not design a homepage and then add a design system. Do not design components and then assemble a page. Design **Mission Studio** first. The homepage, the Storybook, and every future surface are projections of that single application.

This system is organized from the largest structure to the smallest primitive:

```text
Mission Studio
  ↓
Workspace
  ↓
Artifacts
  ↓
Interactions
  ↓
Primitives
```

Buttons, inputs, and badges are derived elements. They exist to serve Mission Studio, not to define it.

---

## 1. UX Principles

### Principle 1 — The user never leaves the application
There is no visual breakpoint where "homepage ends."

### Principle 2 — Context is never lost
Mission / Status / Replay / Governance always remain available.

### Principle 3 — Scrolling changes density, not hierarchy
Expanded → Condensed → Persistent. The information hierarchy stays the same; only the interface density changes.

### Principle 4 — Every state is the same interface
Not three pages. One interface interpolated through scroll progress.

---

## 2. Progressive Collapse

The defining behavior of LDS-002 v3. Mission Studio does not disappear when the user scrolls past it. It compresses through six states:

| State | Height | Sidebar | Content | Notes |
|---|---|---|---|---|
| 01 — Immersive | 100vh | Full labels + dots | Full cards, form inputs | First viewport after hero |
| 02 — Replay | 100vh | Full (✓ completed) | Timeline + evidence | Last full state |
| 03 — Begin Collapse | ~75vh | Narrower | Summary cards | Padding reduced |
| 04 — Compact | ~50vh | Icons only | Single row summary | Labels gone |
| 05 — Sidebar Retracted | ~40vh | Removed | Metadata row | Sidebar slides under |
| 06 — Sticky Bar | 72px | None | Inline chips | Always sticky |

### State transitions

```
100% height
   │
   ▼  (scroll 0–60% of section)
Same workspace, different content: Intent → Discovery → Mission → Expeditions → Governance → Replay
   │
   ▼  (scroll 60–80% — collapse phase)
Workspace height decreases
Sidebar: labels → icons → removed
Cards: full → compact → single row
Padding: generous → tight
   │
   ▼  (scroll 80–100%)
Sticky bar: 72px, no sidebar, mission context only
   │
   ▼  (beyond)
Sticky bar remains at top of viewport
Homepage content scrolls beneath it
```

### Sidebar retraction

```
Full sidebar        Icons only         Removed
┌────────────┐      ┌──────┐           
│ Intent     │      │ ●    │           
│ Discovery  │      │ ●    │    slides under
│ Mission    │  →   │ ●    │  →   content
│ ...        │      │ ●    │           
└────────────┘      └──────┘           
```

Nothing jumps. Nothing fades abruptly. The sidebar physically retracts.

### Content morph

```
Full form/cards     Compact summary     Inline metadata
┌───────────────┐   ┌───────────┐       
│               │   │           │       
│ Textarea      │   │ Mission   │       
│               │   │ Confidence│   Mission • Governed
│               │   │ Replay    │   • Replay Ready • 96%
└───────────────┘   └───────────┘       
```


## 9. Motion

### Duration

| Token | Value | Usage |
|---|---|---|
| `--duration-fast` | 150ms | Hover, small transitions |
| `--duration-base` | 250ms | Phase transitions |
| `--duration-slow` | 400ms | Larger transitions |
| `--duration-collapse` | 500ms | Workspace collapse |

### Easing

| Token | Value | Feel |
|---|---|---|
| `--easing-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | Apple-like, snappy |
| `--easing-in-out` | `cubic-bezier(0.65, 0, 0.35, 1)` | Smooth |
| `--easing-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Springy pop |

### Motion principles

- Use: opacity, scale, translate, height, radius, padding
- Avoid: instant disappearance, hard cuts, opacity-only transitions
- Prefer: FLIP animations, shared element transitions, morphing layouts

## 11. Component Behavior

Mission Studio is stateful across the scroll journey. Rather than remounting new UIs, the same component tree should be preserved to enable shared-element transitions and maintain context.

The collapse should be driven by layout properties: width, height, padding, gap, opacity (secondary content only), border radius, typography scale.

Avoid replacing components with different implementations when a responsive variant can achieve the same result.

### Scroll progress mapping

```text
progress 0.0–0.6:  phase content changes (Intent → → → Replay)
progress 0.6–0.8:  collapse (sidebar retracts, content compresses)
progress 0.8–1.0:  sticky bar forms
progress > 1.0:    sticky bar visible, homepage content scrolls
```

## 12. Layout

### Desktop (>1024px)
- Full three-panel: sidebar (200px) + workspace + footer
- Sidebar shows labels + status dots
- Cards full-width or two-column

### Tablet (768–1024px)
- Sidebar: icons only (labels hidden)
- Mission name hidden in header
- 2-column grids

### Mobile (<768px)
- Sidebar hidden
- Full-width grids
- Single column layouts

## 14. Design Principles Checklist

- [ ] The element maps to a documented SYNTH concept.
- [ ] It uses only tokens defined in this document.
- [ ] It does not introduce arbitrary colors outside the semantic palette.
- [ ] Motion, if present, uses documented duration/easing tokens.
- [ ] It supports progressive disclosure (default state is calm; detail is reachable).
- [ ] It includes a reduced-motion fallback if it animates.
- [ ] Copy uses SYNTH public vocabulary only.
- [ ] It is traceable to an expedition charter or runtime artifact.
- [ ] It is evaluated in the context of the full Mission Studio workspace, not in isolation.
- [ ] It respects the collapse continuum (expanded → compact → sticky).

---

## 15. Conformance

Any deviation from this system must be documented in the expedition charter and accepted as part of the expedition review. The Storybook is the primary conformance artifact: if a component cannot be shown in a Mission Studio workspace state, it does not belong in the public interface.
