# Mission Studio Design System (LDS-002)

> **Design system for the SYNTH Mission Studio Homepage.** Defines the visual language, tokens, and component taxonomy used by every homepage element under EXP-PROGRAM-027. Grounded in the approved homepage design.

---

## Principles

1. **Runtime First**  
   Every visible component maps to a SYNTH concept. No decorative UI exists without a runtime rationale.

2. **Workspace over Pages**  
   The homepage is one persistent workspace. Scroll changes state; it does not load new pages.

3. **Progressive Disclosure**  
   Information density increases as the visitor progresses through Genesis. Begin calm; reveal detail on demand.

4. **Artifact Driven**  
   Replace chat with objects. Everything shown is an artifact that can be inspected, traced, or acted upon.

5. **Calm Computing**  
   Light surfaces, generous whitespace, subtle motion, and clear hierarchy keep the interface readable and deliberate.

## Layout

### Hero workspace

The homepage centers on a single workspace composed of two primary panels:

- **Left panel: Intent Surface**  
  Contains the phase label, intent input, helper text, and source selector.

- **Right panel: Terminal Projection**  
  A read-only terminal surface showing the Agent ↔ SYNTH conversation that results from the selected intent and source.

### Responsive behavior

- **Desktop (>1024px):** panels sit side by side; left panel ~40%, terminal ~60%.
- **Tablet (768–1024px):** panels remain side by side with adjusted ratios.
- **Mobile (<768px):** panels stack; intent surface on top, terminal below.

## Component Taxonomy

Every homepage component maps to a SYNTH concept.

| Component | Concept | Responsibility |
|---|---|---|
| `TopBar` | Public identity | Logo, value tags, primary CTA. |
| `HeroTitle` | Brand projection | Large wordmark and value proposition. |
| `IntentSurface` | Genesis / Intent | Hosts the intent input and source selector. |
| `PhaseLabel` | Genesis phase | Small uppercase label identifying the current phase (e.g., "PHASE 1 — INTENT"). |
| `IntentInput` | Intent artifact | Text input where the visitor describes what they are building. |
| `SourceSelector` | Starting point | Cards for Greenfield, Brownfield, Knowledge, Conversation. |
| `Terminal` | CLI projection | Read-only terminal surface with header, dots, title, and badge. |
| `TerminalHeader` | Runtime status | Title bar with window dots, command title, and governance badge. |
| `AgentColumn` | AI Agent | Column showing the operator's AI agent messages and commands. |
| `SynthColumn` | SYNTH runtime | Column showing SYNTH responses and state. |
| `AgentMessage` | AI Agent utterance | Message bubble from the agent. |
| `SynthMessage` | SYNTH response | Message bubble from SYNTH. |
| `GovernanceBadge` | Governance | Status badge indicating "governed" or other lifecycle states. |
| `ScrollCTA` | Discovery invitation | Link/button inviting the visitor to scroll and see how SYNTH works. |
| `WorkspaceShell` | Mission Workspace | Outer pinned container for later phases (EXP-HOME-002). |
| `GenesisNavigator` | Genesis lifecycle | Vertical phase indicator; shows current lifecycle step. |
| `ArtifactCard` | Artifact | Displays a single artifact (Intent, Discovery, Unknown, Architecture, Mission, Expedition). |
| `ArtifactLineage` | Replay / lineage | Shows how an artifact evolved from prior artifacts. |
| `ConfidenceGate` | Evidence / confidence | Visual indicator of confidence level and missing evidence. |
| `ReplayScrubber` | Replay | Timeline control for scrubbing through artifact history. |
| `CapabilityGrid` | Capability Model | Grid of runtime capabilities linking to documentation. |
| `StatusBar` | Runtime status | Bottom bar showing Replay, Governance, Evidence, and Kernel readiness. |

## Design Principles Checklist

Use this checklist when reviewing any new homepage element.

- [ ] The element maps to a documented SYNTH concept.
- [ ] It uses only tokens defined in this document.
- [ ] It does not introduce arbitrary colors outside the semantic palette.
- [ ] Motion, if present, uses documented duration/easing tokens.
- [ ] It respects the workspace metaphor (no page-like transitions).
- [ ] It supports progressive disclosure (default state is calm; detail is reachable).
- [ ] It includes a reduced-motion fallback if it animates.
- [ ] Copy uses SYNTH public vocabulary only.
- [ ] It is traceable to an expedition charter or runtime artifact.
