## Purpose

Transform SYNTH into an AI-native product.

The human should never need to memorize SYNTH. The AI should.

---

## Problem

Current installation assumes a human CLI operator. SYNTH's primary operator is an AI agent.

---

## Goal

A developer installs SYNTH. Their preferred AI coding assistant immediately knows how to initialize and operate the project.

---

## Deliverables

### Installation

- `npm` package.
- `npx` execution.
- Optional `install.sh`.
- Semantic version verification.

### Agent Integration

Official onboarding guides for:

- Claude Code
- Cursor
- OpenAI Codex
- Gemini CLI
- Windsurf
- Generic MCP-compatible agents (future)

### Agent Prompt Templates

Examples like:

> Initialize this repository with SYNTH.

> Analyze this repository and create a Mission.

> Generate Expeditions.

> Explain the Replay.

> Recover the interrupted Mission.

### Bootstrap Manifest

Agent-readable metadata describing:

- Available commands.
- Capabilities.
- Project layout.
- Public vocabulary.

---

## Acceptance

A new developer can:

1. Install SYNTH.
2. Give their AI access.
3. Tell the AI:

   > Initialize this project with SYNTH.

   without reading architecture documentation.

Specifically:

- Installation completes in under five minutes.
- Agent integration guide exists for at least three major AI assistants.
- Prompt library covers the five core operations listed above.
- Bootstrap manifest is machine-readable.

## Implementation Plan

Approved plan for executing this expedition:

1. **CLI foundation** — Create `src/cli/synth.ts` with commands: `version`, `init`, `govern`, `status`, `mission create`, `expedition create`, `docs generate`, `explain replay`, and `adapter` (delegating to existing adapter CLI). Output structured JSON by default for agent parsing.
2. **Package configuration** — Add `bin` entries (`synth`, `synth-v2`) and `files` whitelist to `package.json`.
3. **Bootstrap manifest** — Design schema in `docs/reference/bootstrap-manifest.md` and generate `.synth/manifest.json` from `synth init`.
4. **install.sh** — Create root installer script checking Node.js and installing the CLI.
5. **Agent guides** — Create `docs/guides/agents/` guides for Claude Code, Cursor, Codex, Gemini CLI, Windsurf, and MCP (future).
6. **Prompt library** — Create `docs/guides/agents/prompts/` with templates for initialize, create-mission, generate-expeditions, explain-replay, and recover-mission.
7. **Tests** — Add `tests/synth-cli.test.js` covering version, init, manifest generation, and JSON output.
8. **Build and verify** — Run `npm run build`, `npm run test:all`, `npm run docs:check-links`, and `npm run audit:repository`.
9. **Expedition update** — Mark this file's Definition of Done and Completion Notes upon completion.
