# EXP-ADOPT-002 — Repository Readiness

**Status:** Proposed  
**Kind:** Expedition  
**Priority:** High  
**Program:** EXP-PROGRAM-037 — Ecosystem Adoption & Community Growth  
**Phase:** I — Platform Foundations  
**Authority:** Synth Architectural Constitution

---

## Goal

Make the SYNTH repository the primary discovery and trust surface for developers.

---

## Purpose

GitHub is the developer hub. The repository must explain what SYNTH is, how to install it, how to contribute, and why the project can be trusted — all within the first minute of reading.

---

## Deliverables

1. **README.md rewrite** with a clear value proposition, install command, quick-start, architecture diagram, and governance summary.
2. **CONTRIBUTING.md alignment** with the current Mission/Expedition governance model.
3. **Issue and pull request templates**.
4. **Code of conduct and license visibility** review.
5. **Release notes template** and CHANGELOG maintenance policy.
6. **Repository metadata**: topics, description, social preview image, and pinned issues.

---

## Acceptance Criteria

- README passes a first-time reader test: an independent reviewer can explain SYNTH in one sentence.
- The install command in README succeeds on all supported platforms.
- All issue and PR templates render correctly.
- The repository has no more than three pinned issues, each with a clear next step.
- A social preview image is present and follows the brand guide.

---

## Out of Scope

- Modifying core governance documents unless required for factual accuracy.
- Homepage content or design.
- Package distribution beyond npm.
