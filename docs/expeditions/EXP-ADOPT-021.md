# EXP-ADOPT-021 — Community Listening & Feedback Loop

**Status:** Proposed  
**Kind:** Expedition  
**Priority:** High  
**Program:** EXP-PROGRAM-037 — Ecosystem Adoption & Community Growth  
**Phase:** IV — Community  
**Authority:** Synth Architectural Constitution

---

## Goal

Create a deterministic feedback loop from community signals to program evidence.

---

## Purpose

Community feedback is only valuable when it becomes actionable evidence. This expedition defines how feedback is collected, triaged, and routed so that questions become docs, bugs become issues, and validated ideas become missions.

---

## Deliverables

1. **Feedback intake channels**: Discord, GitHub discussions, social media, and surveys.
2. **Triage workflow and labeling scheme**.
3. **Feedback-to-evidence mapping**:
   - questions → documentation gaps
   - bugs → issues
   - ideas → mission proposals
4. **Weekly community pulse report template**.
5. **Integration with Mission Studio** for converting validated feedback into missions.

---

## Acceptance Criteria

- Every feedback channel has a documented response-time SLA.
- Triage workflow is published and followed.
- At least one piece of feedback per month becomes a documented evidence item or mission proposal.
- Weekly pulse report is published to maintainers.
- Feedback loop respects privacy and consent.

---

## Out of Scope

- Public feature voting.
- Guaranteeing implementation of every request.
- Real-time sentiment analysis tooling.
