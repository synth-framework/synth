### EXP-VAL-007 — Agentic Mission Lifecycle Correction

**Status:** Completed  
**Kind:** Validation Expedition  
**Priority:** Critical

#### Purpose

Correct the CLI Mission lifecycle to match SYNTH's architecture: creation produces a Mission Draft; approval is a separate, explicit operation.

#### Deliverables

- `src/cli/synth.ts`:
  - `synth mission create` now creates a Planning Session and returns a `MissionDraft` with `draftId`, `confidence`, `unknowns`, `questions`, and `proposals`.
  - New `synth mission approve --draft-id <id>` performs explicit approval and returns a `MissionApprovalDecision`.
  - Business decisions (approval rejected) exit with code `0`; operational failures exit with code `1`.
- `src/api/index.ts`:
  - `approveModel` returns `status: "ok"` with `decision: { approved: false, reason, confidence }` for policy rejections.
  - `status: "error"` is reserved for operational failures.
- `scripts/ai-benchmark.js` updated to run the agentic lifecycle and report `missionDraftCreated` and `missionApprovalDecision`.
- Agent documentation updated to describe the two-step workflow.
- `docs/audits/SYNTH-AI-BENCHMARK-001.md` updated to remove the stale limitation.

#### Architecture Guardrails Maintained

- Mission Studio confidence weights and thresholds were not modified.
- No evidence quality was artificially inflated.
- No Protected Asset was changed.

#### Verification

- `npm run build` — PASS
- `npm run test:ai-benchmark` — PASS
